4. Backend — B1: Login + 2FA (OTP)

Valida el flujo de autenticación con verificación en dos pasos. Cuando un usuario tiene 2FA habilitado, el login devuelve un temp_token; luego se verifica el código OTP almacenado en _otp_store (server-side) para entregar el JWT definitivo.

ec2/api/tests/test_auth.py 
def test_login_with_2fa_returns_temp_token(self, client, mock_dynamodb, db_connection):
    mock_users, _ = mock_dynamodb
    mock_users.query.return_value = {
        'Items': [{'user_id': '2fa-user-id', 'email': 'admin2fa@test.cl',
                  'password_hash': VALID_HASH, 'rol': 'ADMIN'}]
    }
    with patch('routers.auth.send_otp_email') as mock_email:
        response = client.post("/login", json={"email": "admin2fa@test.cl", "password": "testpass123"})
    assert response.status_code == 200
    assert response.json()["two_factor_required"] is True
    assert "temp_token" in response.json()
🧪 Negativo: Código OTP inválido (000000) es rechazado con 401 — último filtro antes del JWT de acceso.

4. Backend — B2: Circuit Breaker

Evalúa el patrón Circuit Breaker para APIs externas (FIRMS, OWM, CONAF). Tras N fallos consecutivos el circuito se abre y ejecuta el fallback sin llamar al servicio real.

ec2/api/tests/test_circuit_breaker.py 
async def test_opens_after_threshold_failures():
    cb = CircuitBreaker("test", failure_threshold=2, recovery_timeout=60.0)
    async def fail(): raise Exception("fail")

    with pytest.raises(Exception): await cb.call(fail)  # fallo 1 → still CLOSED
    assert cb.state == CircuitState.CLOSED

    with pytest.raises(Exception): await cb.call(fail)  # fallo 2 → OPEN
    assert cb.state == CircuitState.OPEN
🧪 Negativo: Circuito abierto rechaza llamadas incluso exitosas hasta que expira el recovery_timeout. Evita cascadas de fallos.
